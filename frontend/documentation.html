Overview

Bob MCP Forge is an AI-powered developer tool that converts plain English descriptions into production-ready Model Context Protocol (MCP) server code. Simply describe what you want your MCP tool to do, and Bob MCP Forge generates complete TypeScript code with built-in compliance checks and one-click deployment to IBM Cloud.

What is MCP?
Model Context Protocol (MCP) is an open protocol that enables AI assistants like Claude to securely connect to external data sources and tools. MCP servers provide tools, resources, and prompts that AI models can use.

Key Features

  • AI-Powered Generation: Describe your tool in plain English and get complete TypeScript code in seconds
  • Compliance Scanning: Automatic security, performance, and policy checks with detailed reports
  • One-Click Fixes: Apply suggested fixes for compliance issues with a single click
  • Cloud Deployment: Deploy to IBM Cloud instantly with automatic containerization
  • Full Code Access: View, edit, and download all generated files
  • MCP Protocol Compliant: Works with Claude and other AI assistants

Getting Started

Follow these steps to create your first MCP server with Bob MCP Forge:

  1. Describe Your Tool
    Navigate to the landing page and enter a description of what you want your MCP server to do. Be specific about the functionality, data sources, and any special requirements.
  2. Select Compliance Profile
    Choose a compliance profile that matches your requirements: General, GDPR, SOC 2, or HIPAA. This ensures your generated code meets specific regulatory standards.
  3. Generate Code
    Click "Generate MCP Server" and wait 15-30 seconds while the AI generates your complete server code, including TypeScript implementation, schema definitions, and Docker configuration.
  4. Review Code
    Browse the generated files in the code preview screen. You can view server.ts, schema.json, package.json, Dockerfile, and README.md.
  5. Check Compliance
    Review the compliance dashboard to see security, performance, and policy issues. Apply suggested fixes for any critical or warning items.
  6. Deploy
    Configure your deployment settings (region, instance size) and click "Deploy to IBM Cloud" to make your MCP server live.
Quick Start

How Bob MCP Forge Works

Bob MCP Forge uses advanced AI to understand your requirements and generate production-ready code. Here's what happens behind the scenes:

1. Understanding Your Request

The AI analyzes your prompt to identify:

  • Required MCP tools and their parameters
  • Data sources and API integrations
  • Authentication and security requirements
  • Error handling and validation needs
  • Performance and caching strategies

2. Code Generation

Based on your requirements, Bob MCP Forge generates:

  • server.ts: Main MCP server implementation with TypeScript
  • schema.json: MCP protocol schema definitions
  • package.json: Node.js dependencies and scripts
  • Dockerfile: Container configuration for deployment
  • README.md: Documentation and usage instructions

3. Compliance Scanning

Every generated server is automatically scanned for:

  • Security Issues: Input validation, API key exposure, authentication
  • Performance: Rate limiting, caching, resource management
  • Code Quality: TypeScript strict mode, error handling, logging
  • Policy Compliance: GDPR, SOC 2, HIPAA requirements (based on selected profile)

4. Deployment Preparation

Bob MCP Forge prepares your server for deployment by:

  • Building a Docker container image
  • Configuring environment variables
  • Setting up health checks
  • Provisioning IBM Cloud resources

Compliance Checks

Bob MCP Forge performs comprehensive compliance scanning on all generated code. Issues are categorized by severity:

CRITICAL Critical Issues

Security vulnerabilities or compliance violations that must be fixed before deployment. Examples: missing input validation, exposed API keys, authentication bypasses.

WARNING Warning Issues

Best practice violations or potential problems that should be addressed. Examples: missing error handling, no rate limiting, inadequate logging.

INFO Info Issues

Suggestions for improvement or optional enhancements. Examples: TypeScript strict mode, structured logging, additional documentation.

Common Compliance Issues

  • Missing Input Validation: User inputs not validated against schema
  • API Key Exposure: Credentials hardcoded in source code
  • Missing Error Handling: API calls not wrapped in try-catch blocks
  • No Rate Limiting: Unlimited API requests allowed
  • Inadequate Logging: No structured logging for debugging

Applying Fixes

For each issue, Bob MCP Forge provides:

  • Detailed explanation of the problem
  • File location and line numbers
  • Suggested fix with code examples
  • "Apply Fix" button for one-click resolution

Generated File Structure

Every MCP server generated by Bob MCP Forge includes these files:

server.ts

Main TypeScript implementation of your MCP server. Includes:

import { Server } from '@modelcontextprotocol/sdk/server/index.js'; import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js'; // Server initialization const server = new Server({ name: 'your-mcp-server', version: '1.0.0' }, { capabilities: { tools: {} } }); // Tool definitions and handlers server.setRequestHandler(ListToolsRequestSchema, async () => ({ tools: [/* your tools */] }));

schema.json

MCP protocol schema defining your tools, resources, and prompts:

{ "name": "your-mcp-server", "version": "1.0.0", "tools": [ { "name": "tool_name", "description": "Tool description", "inputSchema": { /* JSON Schema */ } } ] }

package.json

Node.js package configuration with dependencies and scripts:

{ "name": "your-mcp-server", "version": "1.0.0", "type": "module", "dependencies": { "@modelcontextprotocol/sdk": "^1.0.0" }, "scripts": { "build": "tsc", "start": "node dist/server.js" } }

Dockerfile

Container configuration for deployment:

FROM node:20-alpine WORKDIR /app COPY package*.json ./ RUN npm install COPY . . RUN npm run build CMD ["npm", "start"]

README.md

Documentation including setup instructions, usage examples, and API reference.

Deployment

Bob MCP Forge makes deployment to IBM Cloud simple and fast. The deployment process includes:

Pre-Deployment Checklist

Before deployment, Bob MCP Forge verifies:

  • ✅ Code successfully generated
  • ✅ All dependencies validated
  • ✅ Dockerfile created
  • ✅ Environment variables configured
  • ✅ No critical compliance issues

Configuration Options

  • Region: Choose from us-south, us-east, eu-gb, or eu-de
  • Instance Size: Small (1 vCPU, 2GB), Medium (2 vCPU, 4GB), or Large (4 vCPU, 8GB)
  • Server Name: Custom name for your deployment

Deployment Process

  1. Building Docker image from your code
  2. Pushing image to IBM Cloud Registry
  3. Creating Cloud Foundry application
  4. Deploying application to selected region
  5. Running health checks
  6. Providing live URL

Typical deployment time: 2-3 minutes

Post-Deployment

After successful deployment, you'll receive:

  • Live URL for your MCP server
  • Status indicators (running, region, instance)
  • Access to deployment logs
  • Options to create new servers or view dashboard

Troubleshooting

Generation Issues

Problem: Generation takes too long or times out
Solution: Simplify your prompt or break it into smaller components. Try being more specific about the core functionality.
Problem: Generated code doesn't match requirements
Solution: Provide more detailed specifications in your prompt. Include specific API endpoints, data formats, and error handling requirements.

Compliance Issues

Problem: Too many critical compliance issues
Solution: Use the "Apply Fix" buttons to automatically resolve issues. Review the suggested fixes and apply them one by one.
Problem: Can't deploy due to compliance failures
Solution: All critical issues must be resolved before deployment. Focus on security issues first (input validation, API key exposure).

Deployment Issues

Problem: Deployment fails during build
Solution: Check the deployment logs for specific errors. Common issues include missing dependencies or invalid Dockerfile configuration.
Problem: Server not responding after deployment
Solution: Verify environment variables are set correctly. Check health check logs in the deployment dashboard.
