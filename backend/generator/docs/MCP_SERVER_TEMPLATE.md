# MCP Server Template Documentation

This document describes the structure and logic of the MCP servers generated by this module.

---

## MCP Architecture

The generator emits a complete TypeScript project based on the official Model Context Protocol (MCP) standards.

**Core Principles:**
- **Standardization**: Uses the `@modelcontextprotocol/sdk`.
- **Type Safety**: Strictly typed with TypeScript and Zod.
- **Portability**: Uses `stdio` transport for universal client compatibility.
- **Resilience**: Implements timeouts and robust error handling.

---

## Example Generated Tool: Customer Lookup

The standard template generates a tool for retrieving customer information.

**Handled Logic:**
- Input validation (e.g., email format).
- Simulated database lookup with asynchronous delay.
- Error wrapping for MCP-compliant responses.

---

## Generated File Specifications

### `package.json`
Configures the project as an ES Module.
- **Main Dependencies**: `@modelcontextprotocol/sdk`, `zod`.
- **Scripts**: `build`, `start`, `dev`.

### `src/server.ts`
The main entry point. It initializes the `McpServer` and registers tools with their respective Zod schemas and implementation handlers.

### `.env.example`
Provides a template for environment-specific configuration:
- `LOG_LEVEL`: Control logging verbosity.
- `TIMEOUT_MS`: Set maximum execution time for tools.

### `Dockerfile`
A multi-stage build for creating small, production-ready container images.

### `manifest.json`
Metadata about the server, tools, and resources exposed to the MCP host.

---

## Extending the Template

The generator uses a modular template system. To add new features to all generated servers, modify the base templates in `src/templates/`.

**Recommended Extensions:**
- Adding authentication headers to outgoing requests.
- Integrating with specific database drivers (e.g., Prisma, Mongoose).
- Supporting additional MCP transports like SSE.
